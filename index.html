Race Ready

PWA / running / AI-assisted workflow

Race Ready is a mobile-first race-prep app for recreational half marathon and marathon runners. The initial audience is first-time or less-experienced long-distance runners who want more guidance when training gets disrupted by missed runs, soreness, fatigue, travel, or uncertainty.

I’m building the MVP as a PWA first so I can validate the core experience before investing in a native iOS app. The near-term goal is to test whether a simple check-in, run logging flow, and weekly review can help runners feel more prepared during training.

Core product loop

  • Sign in with Google
  • Choose or manually enter a race
  • Assess race readiness
  • Generate a simple initial plan
  • Check in before runs
  • Log completed runs
  • Get missed-workout guidance
  • Receive a weekly review

How the LLM fits

  • Extracts messy run logs into structured data
  • Drafts workout briefings
  • Explains missed-workout adjustments
  • Generates weekly review summaries
  • Stays behind server-side rules and safety checks

Architecture direction

The UI is intentionally thin: it displays state and submits actions. Race-distance rules, safety guardrails, plan viability, quotas, and LLM orchestration live in the server/domain layer. The goal is to keep the core logic reusable if the product later moves from PWA to native iOS.

Implementation notes
